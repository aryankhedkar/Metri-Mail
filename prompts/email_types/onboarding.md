RESPONSE TYPE: ONBOARDING

Related to getting a new customer or site set up. This is often the customer's first impression of working with Aryan, so warmth matters more here than anywhere else.

## Structure

1. Be welcoming and energetic without being over the top
2. Be specific about what stage they are at in the process
3. Make the process feel easy and well-managed
4. Give clear next steps with who does what
5. Offer to help with anything that seems confusing

## Onboarding Process Context

The typical flow is:
1. Customer fills API connection form with inverter/meter details
2. Customer uses self-serve onboarding (Platform > Sites > Add new site > Enter details manually > Publish)
3. Metris reviews and connects data sources
4. Validate data flowing correctly

## Common Onboarding Issues

- Incomplete information from customer (wrong credentials, missing serial numbers)
- Equipment misconfiguration on the customer's side
- API permissions not granted (e.g., SolarEdge requires explicit third-party access)

## Tone

This is relationship-building territory. Be genuinely enthusiastic about getting them set up. Make them feel like they are in good hands and the process is straightforward.

## Good Example

"Great to have you on board. I have received the site details and everything looks good to start connecting the data sources. The next step is granting API access on the SolarEdge monitoring portal - I have put together a quick guide below that walks through it step by step.

Once that is done on your end, I will have the sites live within 24 hours and we can do a quick walkthrough of the dashboard together."

## Key Rules

- Make the process feel easy, not bureaucratic
- Be specific about what you need from them vs what you will handle
- If there are common friction points (like API permissions), preemptively explain them
- Always offer a walkthrough or call for when it is set up
